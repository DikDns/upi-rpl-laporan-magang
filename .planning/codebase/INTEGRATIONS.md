# External Integrations

**Analysis Date:** 2026-06-07

## APIs & External Services

**None (fully offline):**
- No external API calls at runtime. All document generation runs locally via Python scripts.
- No network requests during skill execution (after install).

**Install-time only:**
- GitHub archive download — `https://github.com/dikdns/upi-rpl-laporan-magang/archive/refs/heads/main.tar.gz`
  - Used by: `install.sh` (curl) and `install.ps1` (Invoke-WebRequest)
  - Auth: None (public repository)
  - Only invoked during `./install.sh` or remote install; never during skill runtime.

## Data Storage

**Databases:**
- None. No database is used.

**File Storage:**
- Local filesystem only
  - Config: `~/.claude/magang-tools/config.json` — student/program configuration generated by `/init` skill
  - Scripts: `~/.claude/magang-tools/scripts/*.py`
  - Templates: `~/.claude/magang-tools/templates/pks_template.docx`
  - Assets: `~/.claude/magang-tools/assets/upi-logo.png`
  - Generated skills: `~/.claude/magang-tools/generated-skills/` (per-bab SKILL.md files)
  - Plugin cache: `~/.claude/plugins/cache/rpl-magang/`
  - Output DOCX files: written to user-specified paths (e.g., `Laporan.docx`, `Logbook.docx`)

**Caching:**
- None beyond local plugin cache managed by Claude Code's plugin system.

## Authentication & Identity

**Auth Provider:**
- None. No authentication required for any runtime operation.
- Plugin registration into Claude Code's `~/.claude/settings.json` uses local file writes (no API keys or tokens).

## Monitoring & Observability

**Error Tracking:**
- None. Script-level exit codes only (0=success, 1=file not found, 2=parse failed).
- `init_pedoman.py` documents exit codes explicitly:
  - Exit 0: success
  - Exit 1: PDF file not found
  - Exit 2: parse failed (caller uses fallback config)

**Logs:**
- No structured logging. Scripts print to stdout/stderr directly.
- Installer scripts use colored terminal output (ANSI escape codes) for status messages.

## CI/CD & Deployment

**Hosting:**
- GitHub: `https://github.com/dikdns/upi-rpl-laporan-magang` (public repo, `dikdns` user)
- Distribution: Direct archive download via curl; no npm/pypi/homebrew packaging.

**CI Pipeline:**
- Not detected. No `.github/workflows/` directory found.

## Environment Configuration

**Required env vars:**
- None at runtime. All configuration read from `~/.claude/magang-tools/config.json`.

**Secrets location:**
- None. No secrets, API keys, or credentials used.

## Webhooks & Callbacks

**Incoming:**
- None.

**Outgoing:**
- None.

## Claude Code Plugin System Integration

**Plugin registration files (local, not external):**
- `~/.claude/plugins/installed_plugins.json` — install record
- `~/.claude/plugins/known_marketplaces.json` — marketplace source registry
- `~/.claude/settings.json` — `enabledPlugins` and `extraKnownMarketplaces` gates

**Plugin manifest files (in-repo):**
- `.claude-plugin/plugin.json` — plugin identity (`rpl-magang`, version `1.0.0-beta.2`)
- `.claude-plugin/marketplace.json` — Claude Code marketplace schema (`anthropic.com/claude-code/marketplace.schema.json`)

**Skills (Claude Code slash-commands registered as plugin skills):**
- `/init` — `skills/init/SKILL.md` — parse pedoman PDF, generate config
- `/laporan` — `skills/laporan/SKILL.md` — draft laporan bab sections
- `/laporan-compile` — `skills/laporan-compile/SKILL.md` — compile bab markdown to DOCX via `generate_laporan.py`
- `/logbook` — `skills/logbook/SKILL.md` — generate logbook DOCX via `generate_logbook.py`
- `/pks` — `skills/pks/SKILL.md` — generate PKS DOCX via `generate_pks.py`
- `/penilaian-penyelia` — `skills/penilaian-penyelia/SKILL.md` — generate penilaian form DOCX via `generate_penilaian.py`
- Generated per-bab skills — `~/.claude/magang-tools/generated-skills/bab*/SKILL.md` — runtime-generated by `generate_bab_skills.py`
