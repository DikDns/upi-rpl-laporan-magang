# Technology Stack

**Analysis Date:** 2026-06-07

## Languages

**Primary:**
- Python 3 (>=3.11 inferred from type hint syntax `list[dict] | None`) - All document generation scripts and PDF parsing
- Bash - Installer scripts (`install.sh`, `uninstall.sh`)
- PowerShell - Windows installer (`install.ps1`)
- Markdown - Skill definitions (`skills/*/SKILL.md`), documentation, and laporan draft content

**Secondary:**
- JSON - Config format (`~/.claude/magang-tools/config.json`), plugin manifests

## Runtime

**Environment:**
- Python 3 (system or via venv) — `~/.claude/magang-tools/venv/` isolated per-user install
- Claude Code (Anthropic CLI) — runtime host for skill execution; skills invoke Python scripts via Bash tool

**Package Manager:**
- pip (standard) — installs into isolated venv
- Lockfile: Not present (only `requirements.txt` with version lower bounds)

## Frameworks

**Core:**
- python-docx (>=1.1.2) — DOCX generation for all document types (laporan, logbook, PKS, penilaian)
- pdfplumber (>=0.11.0) — PDF text extraction for pedoman/config init
- Pillow (>=10.0.0) — Image processing/embedding for laporan gambar captions

**Claude Code Plugin System:**
- Plugin format: `.claude-plugin/plugin.json` + `skills/*/SKILL.md`
- Marketplace: `.claude-plugin/marketplace.json`
- Skills are Markdown prompt files that orchestrate Bash + Read + Write tools within Claude Code

**Testing:**
- Python `unittest` / script-level tests — `tests/test_generate_bab_skills.py`, `tests/test_generate_laporan_toc.py`
- No external test framework (pytest not required)

**Build/Dev:**
- No build step — pure Python scripts, no compilation
- venv setup handled by `install.sh` / `install.ps1`

## Key Dependencies

**Critical:**
- `python-docx>=1.1.2` — Core output format; all generated artifacts are `.docx` files opened in Microsoft Word / LibreOffice
- `pdfplumber>=0.11.0` — Required for `/init` skill to parse pedoman PDF and extract student configuration
- `Pillow>=10.0.0` — Required for embedding images in laporan bab sections with auto-numbered captions

**Infrastructure:**
- Python venv at `~/.claude/magang-tools/venv/` — isolates dependencies from system Python

## Configuration

**Environment:**
- No `.env` files; configuration stored in `~/.claude/magang-tools/config.json` (generated by `/init` skill from pedoman PDF)
- Config keys include: student name, NIM, program studi, company name, supervisor info, document formatting params (font, margins, page size)

**Build:**
- `requirements.txt` — pip dependency spec
- `scripts/settings.local.json` — local dev overrides (dev-only, not installed)
- `.claude-plugin/plugin.json` — plugin identity and version
- `.claude-plugin/marketplace.json` — Claude Code marketplace registration schema

## Platform Requirements

**Development:**
- macOS or Linux (bash available); Windows via PowerShell with `install.ps1`
- Python 3 installed (system-level, pre-venv)
- `curl` installed (for remote install from GitHub)
- Claude Code CLI installed

**Production:**
- Deployed as a Claude Code plugin to `~/.claude/plugins/cache/rpl-magang/`
- Python scripts copied to `~/.claude/magang-tools/scripts/`
- Templates copied to `~/.claude/magang-tools/templates/`
- Assets (UPI logo PNG) copied to `~/.claude/magang-tools/assets/`
- GitHub: `https://github.com/dikdns/upi-rpl-laporan-magang` (archive install via curl)
