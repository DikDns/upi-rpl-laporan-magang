## Directory Purposes

**`skills/`:**
- Purpose: Claude Code slash command implementations — these are AI instruction files, not code
- Contains: One subdirectory per skill, each with a SKILL.md
- Key files: `skills/init/SKILL.md`, `skills/laporan/SKILL.md`, `skills/logbook/SKILL.md`
- Note: Skill names map to slash commands as `/rpl-magang:{skill-name}`

**`scripts/`:**
- Purpose: Python document generators — the actual DOCX/config production layer
- Contains: Python 3 scripts; all standalone executables with argparse CLI
- Key file: `scripts/generate_laporan.py` (most complex, 704 lines — handles full laporan DOCX rendering)
- Note: Installed to `~/.claude/magang-tools/scripts/` and called via venv python

**`templates/`:**
- Purpose: Fixed-content DOCX base files where only placeholder tokens change
- Contains: `pks_template.docx` — the official UPI PKS agreement with yellow-highlighted partner fields
- Note: User can supply a custom PKS template path via `pks_template_path` in config

**`assets/`:**
- Purpose: Static image files embedded in generated documents
- Contains: `upi-logo.png` — used in laporan cover page
- Note: Installer copies to `~/.claude/magang-tools/assets/`; cover generation falls back to "LOGO UPI" text placeholder if absent

**`tests/`:**
- Purpose: Python unit tests for script logic
- Contains: `test_generate_bab_skills.py`, `test_generate_laporan_toc.py`
- Note: Run with pytest; `.pytest_cache/` is gitignored

**`.claude/`:**
- Purpose: Claude Code plugin registration metadata
- Contains: `marketplace.json` (public plugin identity), `settings.local.json` (local, gitignored)
- Generated: No — hand-authored

**`.claude-plugin/`:**
- Purpose: Plugin descriptor read by Claude Code plugin system
- Contains: `plugin.json` (name: "rpl-magang", version, author, keywords)

## Key File Locations

**Entry Points (skill instructions):**
- `skills/init/SKILL.md`: One-time setup — run first
- `skills/laporan/SKILL.md`: Front-matter section authoring
- `skills/laporan-compile/SKILL.md`: Compile all sections to DOCX
- `skills/logbook/SKILL.md`: Logbook generation (single/batch/import)
- `skills/penilaian-penyelia/SKILL.md`: Assessment form generation
- `skills/pks/SKILL.md`: PKS agreement generation

**Core Python Scripts:**
- `scripts/generate_laporan.py`: Most complex script — markdown-to-DOCX engine, image embedding, table rendering, cover/pengesahan templates
- `scripts/init_pedoman.py`: PDF parsing engine using pdfplumber, config schema definition
- `scripts/generate_bab_skills.py`: Dynamic skill generation — produces per-bab SKILL.md files from config

**Plugin Identity:**
- `.claude-plugin/plugin.json`: Plugin name/version for Claude Code
- `.claude/marketplace.json`: Public marketplace metadata

**Installation:**
- `install.sh`: macOS/Linux — downloads repo, installs venv, copies files, registers plugin
- `install.ps1`: Windows equivalent
- `requirements.txt`: Python dependencies (python-docx, pdfplumber, Pillow)

**Testing:**
- `tests/test_generate_bab_skills.py`: Tests for dynamic skill generation
- `tests/test_generate_laporan_toc.py`: Tests for TOC/section ordering

## Naming Conventions

**Files:**
- Skills: `SKILL.md` (uppercase, always this filename)
- Python scripts: `generate_{document_type}.py` for generators, `{action}_{subject}.py` for utilities
- DOCX output files: `{DocumentType}_{SubjectName}.docx` (e.g., `Logbook_NamaMhs_Minggu1.docx`, `PKS_CompanyName.docx`)
- Draft markdown files: `{section-name}.md` using kebab-case (e.g., `kata-pengantar.md`, `lembar-pengesahan.md`, `bab1.md`)

**Directories:**
- Skill dirs: kebab-case matching slash command name (e.g., `laporan-compile/`, `penilaian-penyelia/`)
- Script naming: snake_case Python convention

**Config keys:**
- Snake_case throughout config.json (e.g., `student_identity`, `file_naming`, `penilaian_penyelia`)

## Where to Add New Code

**New skill (new slash command):**
- Create: `skills/{skill-name}/SKILL.md`
- Pattern: Follow existing SKILL.md structure — YAML frontmatter + `<objective>`, `<constants>`, `<steps>` XML blocks
- Ensure: Add skill name to plugin.json or install script if it needs separate registration

**New Python document generator:**
- Create: `scripts/generate_{type}.py`
- Pattern: argparse CLI, reads config via `load_config()`, outputs JSON result to stdout, uses `versioned_path()` for output files
- Call convention: `~/.claude/magang-tools/venv/bin/python scripts/generate_{type}.py --data /tmp/{type}_data.json --output path.docx`

**New static template:**
- Place: `templates/{template_name}.docx`
- Reference: Add path to config.json or hardcode in corresponding script's constants

**New test:**
- Place: `tests/test_{script_name}.py`
- Framework: pytest
- Run: `pytest tests/` from repo root (requires venv with test dependencies)

**New config field:**
- Define default: In `scripts/init_pedoman.py` (alongside `FILE_NAMING_DEFAULT`, `PENILAIAN_DEFAULT`)
- Extract from PDF: Add extractor function in `init_pedoman.py`
- Consume: Read from `config.json` in the relevant script's `load_config()` call

## Special Directories

**`.planning/`:**
- Purpose: GSD planning documents (phase plans, codebase maps)
- Generated: Yes (by GSD commands)
- Committed: Yes

**`.pytest_cache/`:**
- Purpose: pytest cache
- Generated: Yes
- Committed: No (gitignored)

**`~/.claude/magang-tools/generated-skills/`:**
- Purpose: Runtime-generated per-bab SKILL.md files created after init from user's pedoman
- Generated: Yes (by `generate_bab_skills.py` during init)
- Committed: No (lives in user's home dir, not repo)

**`~/.claude/plugins/cache/`:**
- Purpose: Claude Code's plugin loading cache — skills here are available as slash commands
- Generated: Yes (populated by installer and skill sync step in init)
- Committed: No
