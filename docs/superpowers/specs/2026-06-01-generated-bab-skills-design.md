# Generated Per-Bab Skills + DAFTAR ISI + Page Numbering Design

## Goal

Replace the `laporan` monolith's bab steps and compile step with:
1. Per-bab skills generated by `init` from the student's pedoman config — lighter context, verbatim section titles
2. A static `laporan-compile` skill
3. Auto-generated DAFTAR ISI (always), DAFTAR TABEL/GAMBAR (conditional) at compile time
4. Correct DOCX page numbering: no number on cover, Roman (i, ii...) on front matter, Arabic (1, 2...) from BAB I onward

## Architecture

### Data flow

```
/rpl-magang:init
  └─ parses pedoman PDF → config.json
  └─ NEW: generates per-bab SKILL.md files
       source: ~/.claude/magang-tools/generated-skills/
       sync:   ~/.claude/plugins/cache/rpl-magang/.../skills/
  └─ tells user to restart Claude Code

after restart:
  /rpl-magang:laporan-bab-1   ← generated, pedoman-specific
  /rpl-magang:laporan-bab-2
  /rpl-magang:laporan-bab-3
  /rpl-magang:laporan-bab-4
  /rpl-magang:laporan-compile ← static, in plugin repo
  /rpl-magang:laporan         ← trimmed: cover, lembar-pengesahan, kata-pengantar only
```

### File locations

| Path | Role |
|------|------|
| `~/.claude/magang-tools/generated-skills/laporan-bab-N/SKILL.md` | Persistent source (survives plugin updates) |
| `~/.claude/plugins/cache/rpl-magang/rpl-magang/0.1.0/skills/laporan-bab-N/SKILL.md` | Claude Code discovery path |
| `skills/laporan-compile/SKILL.md` | Static, shipped in plugin repo |
| `scripts/generate_laporan.py` | DOCX compile script — gets new renderers + section breaks |
| `install.sh` | Syncs generated-skills to cache after plugin update |

## Component Specs

### 1. Generated per-bab SKILL.md

Init renders one `SKILL.md` per bab from `config.document_structure.bagian_isi`.

Template structure (example: babIII → laporan-bab-3):

```markdown
---
name: laporan-bab-3
description: "Tulis BAB III [title] — [section 3.1 title], [3.2 title], ..."
allowed-tools:
  - Read
  - Write
  - Bash
  - AskUserQuestion
---

<objective>
Tulis BAB III laporan magang berdasarkan pedoman. Simpan sebagai bab3.md.
</objective>

<constants>
CONFIG_PATH = ~/.claude/magang-tools/config.json
OUTPUT_DIR  = ./laporan-draft/ (or from --output-dir arg)
</constants>

<ai-writing-rule>
[identical to laporan skill]
</ai-writing-rule>

<rpl-emphasis-rule>
[identical to laporan skill]
</rpl-emphasis-rule>

<image-rule>
[identical to laporan skill — only prompted in bab3]
</image-rule>

<steps>

## Step 1 — Check config and output dir

Read ~/.claude/magang-tools/config.json. If missing → "Jalankan /rpl-magang:init dulu."
Ask output dir if not in $ARGUMENTS (default: ./laporan-draft/).

## Step 2 — BAB III sub-sections dari pedoman kamu

3.1 [verbatim title from config]
3.2 [verbatim title from config]
...

[writing hints table — keyword-matched format guidance]

[bab3 only: sections_flexible check — ask role type, adapt if non-coding]

[bab3 only: image-rule prompt at end]

## Step 3 — Save and confirm

Write bab3.md. Show progress checklist from output dir.
</steps>
```

**description field** includes verbatim sub-section titles so tab completion previews what's inside.

**ai-writing-rule, rpl-emphasis-rule, image-rule** are inlined into each generated skill (no cross-skill include mechanism exists).

**image-rule prompt** only included in babIII skill (most relevant; other babs may still reference it but it's not the primary focus).

**Bab III special rule** (when `sections_flexible: true` in config): generated skill includes the role-type question and non-coding adaptation note.

### 2. Init skill changes

New **Step 8 — Generate per-bab skills** inserted between Step 6 (save config) and Step 7 (confirm):

1. Read `config.document_structure.bagian_isi` — all bab keys and sections
2. Map keys: `babI → bab1`, `babII → bab2`, `babIII → bab3`, `babIV → bab4`, etc.
3. For each bab: render SKILL.md string with verbatim sub-section titles from config
4. Write to `~/.claude/magang-tools/generated-skills/laporan-bab-N/SKILL.md`
5. Sync bab skills to plugin cache:
```bash
cp -r ~/.claude/magang-tools/generated-skills/laporan-bab-* \
       ~/.claude/plugins/cache/rpl-magang/rpl-magang/0.1.0/skills/
```

`laporan-compile` is a static skill in the plugin repo — already in cache, not generated.

Step 7 (confirm) updated to add:
```
⚠️ Restart Claude Code untuk mengaktifkan skill laporan-bab-N yang baru di-generate.
```

### 3. laporan monolith — trimmed

Remove: Step 5 (bab content), Step 2 COMPILE mode.

Keep: Step 3 (COVER), Step 3b (LEMBAR PENGESAHAN), Step 4 (KATA PENGANTAR).

Step 1 argument list updated: `cover`, `lembar-pengesahan`, `kata-pengantar` only.

`argument-hint` updated to reflect trimmed surface.

### 4. laporan-compile skill (static, in repo)

`skills/laporan-compile/SKILL.md`:

```markdown
---
name: laporan-compile
description: "Compile semua section .md jadi satu DOCX laporan — DAFTAR ISI, page numbering otomatis"
allowed-tools:
  - Read
  - Write
  - Bash
  - AskUserQuestion
---
```

Steps:
1. Read config. Ask output dir (default `./laporan-draft/`).
2. Check required sections from `config.document_structure.bagian_awal` — warn on missing `.md` files.
3. Run `generate_laporan.py --sections-dir [dir] --output [file_naming.laporan from config]`.
4. Confirm output path. Remind: open in Word → click "Update Table" to finalize DAFTAR ISI page numbers.

### 5. generate_laporan.py — new compile features

#### 5a. DAFTAR ISI — Word TOC field

Always inserted at the `daftar-isi` position in `SECTIONS_ORDER`.

Implementation: inject `w:fldChar` + `w:instrText` XML with instruction:
```
TOC \o "1-3" \h \z \u
```
- `\o "1-3"` — include heading levels 1–3 (BAB, x.y, x.y.z)
- `\h` — hyperlinks
- `\z` — hide tab leader in web view
- `\u` — use applied paragraph outline level

Title "DAFTAR ISI" rendered as H1 (centered, bold, chapter size) above the field. Student opens DOCX → Word prompts "Update Table" → dot leaders, page numbers, hyperlinks populate automatically.

#### 5b. DAFTAR TABEL — conditional

Before compile, scan all bab `.md` files for markdown table syntax (`|---|`).

If tables found → insert DAFTAR TABEL page at `daftar-tabel` position with TOC field:
```
TOC \h \z \c "Tabel"
```
If no tables → skip, do not insert empty page.

#### 5c. DAFTAR GAMBAR — conditional

Before compile, scan all bab `.md` files for image syntax (`![`).

If images found → insert DAFTAR GAMBAR page at `daftar-gambar` position with TOC field:
```
TOC \h \z \c "Gambar"
```
If no images → skip.

#### 5d. Section breaks + page numbering

`compile_sections()` inserts `w:sectPr` XML at two section boundaries:

| Section | Pages | Footer style |
|---------|-------|--------------|
| 1 | Cover only | `w:pgNumType w:fmt="none"` — no footer |
| 2 | Lembar Pengesahan → last Daftar page | `w:pgNumType w:fmt="lowerRoman" w:start="1"` — i, ii, iii... |
| 3 | BAB I → end (Lampiran) | `w:pgNumType w:fmt="decimal" w:start="1"` — 1, 2, 3... |

Section 2 starts: page after cover (first page of `lembar-pengesahan`).
Section 3 starts: first page of `bab1`.

Footer paragraph added to section 2 and 3: centered, body font, page number field (`w:fldChar` with `PAGE`).

### 6. install.sh — generated-skills sync

Add after plugin cache population:

```bash
GENERATED="$HOME/.claude/magang-tools/generated-skills"
CACHE="$HOME/.claude/plugins/cache/rpl-magang/rpl-magang/0.1.0/skills"

if [ -d "$GENERATED" ]; then
  echo "Syncing generated bab skills to plugin cache..."
  for skill_dir in "$GENERATED"/laporan-bab-*/; do
    [ -d "$skill_dir" ] && cp -r "$skill_dir" "$CACHE/"
  done
fi
```

This runs on every `install.sh` execution — so after any plugin update, generated skills are automatically restored without re-running init.

## Constraints

- Plugin cache path `rpl-magang/0.1.0` is hardcoded in sync commands. If version bumps, init and install.sh must update the path. Acceptable for now — version bump is a developer action, not student action.
- DAFTAR ISI page numbers require student to click "Update Table" in Word. This is a known Word behavior, not a bug. The skill confirm step reminds them.
- Generated skills inherit `ai-writing-rule`, `rpl-emphasis-rule`, `image-rule` by inlining — if those rules change in the future, init must be re-run to regenerate.
- `sections_flexible: true` in config means bab3 sub-sections are pedoman defaults, not fixed. The generated bab3 skill includes a note to ask about role type and adapt accordingly.

## What Does NOT Change

- `generate_logbook.py`, `generate_penilaian.py`, `generate_pks.py` — untouched
- `logbook`, `pks`, `penilaian-penyelia` skills — untouched
- `init` PDF parsing and config extraction — untouched
- DOCX formatting (font, margins, heading sizes) — untouched
